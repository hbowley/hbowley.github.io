Real-Time Disaster Response Analytics

Built a Python and R-based automated pipeline using Selenium and Google Sheets API to track utility outages, extreme heat, and boil water notices during Hurricane Milton for Catalyst Miami / CEFA-Clean Energy for All.

Impact:

  • Centralized real-time outage data used to activate community response teams
  • Enhanced disaster relief coordination across Miami-Dade County
  • Scalable framework for future emergency response deployments

How it works:

  • Selenium scrapes live utility outage data at regular intervals
  • Data flows through Google Sheets API for real-time team access
  • R scripts generate summary analytics and trend reports
  • Automated alerts when outage thresholds are exceeded

Tech: Python, R, Selenium, Google Sheets API, Automation

Store Performance Tracking System

Built for Abcaco Foods through Portable Networks — a production Shiny application deployed on AWS with a PostgreSQL backend that enables real-time sales data entry for field teams.

What it does:

  • Real-time sales data entry and tracking for field representatives
  • Product placement monitoring across store locations
  • Manager engagement dashboards
  • Live analytics on store performance trends

Architecture:

  • Shiny frontend deployed on AWS EC2
  • PostgreSQL (RDS) backend for persistent data storage
  • Role-based access for field teams vs. management
  • Automated reporting and export functionality

Tech: R, Shiny, AWS (EC2, RDS), PostgreSQL, Production Deployment

Highlights:

  • Publication-quality figures with ggplot2
  • Contributed to peer-reviewed publications and conference presentations

Tech: R, tidyverse, ggplot2, Statistical Modeling

👁️ SRCD Conference Poster

Created a fully reproducible conference poster for the Society for Research in Child Development (SRCD) using the posterdown R package. No PowerPoint, no manual layout — everything from the data analysis to the final poster is generated from a single R Markdown file.

Why this matters:

  • Entire poster is version-controlled and reproducible
  • Update the data → poster updates automatically
  • Demonstrates that academic publishing can be code-driven

Tech: R, posterdown, R Markdown, Academic Publishing
